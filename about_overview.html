About - Overview

MUTIC is a experimental language for generating repeating and/or changing musical patterns and it can be used for making music, and exploring music theory.

Patterns can be expressed using conventional music notation and are based on changes in pitch, duration and velocity. Generators are chosen from a set of finite-state machines that produce either constant or evolving sequences.

There are currently generators for simple pitch, duration and velocity changes, and percussive types that use binary division and euclidean algorithms. Generators can also be used to create sequences of MIDI control codes rather than musical notes. Genetic Algorithms, L-Systems and Markov Chains are options for sequence generation in the future.

Generators are chained together in tracks to be played through the selected sound devices. These can be a combination of SoundFont synthesisers, third-party software sound engines via ALSA, and external hardware MIDI devices such as keyboards and samplers. Tracks can be grouped hierarchically for larger compositions. Parameterisation of the generators allows for real-time control via input from a MIDI controller for use in a live performance setting.

The interpreter is written entirely in C on LINUX and is small and fast. It makes low demands on system resources and can be run on modest hardware, such as the Raspberry Pi. The syntax is LISP-like with many of the operators overloaded to keep the language as simple as possible. The architecure uses functional/object paradigms and is easy to extend. XML and HTML can be included in the code, and patterns can be output as images for display.

Symbol libraries can be imported, and these include basic music theory with definitions for notes, chords, scales and progressions. The percussive libraries have 64 step standard and ethnic patterns, and genre-based kit definitions. There are also MIDI mappings for various synthesisers, drum machines and samplers.
